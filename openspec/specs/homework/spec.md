# 课程作业功能需求规范

### Requirement: 作业列表差异化展示
- **GIVEN** 用户访问课程作业页
- **WHEN** 用户角色为教师
- **THEN** 筛选条显示“已发布”、“未发布”；不显示进度条；显示批改/提交统计。
- **WHEN** 用户角色为学生
- **THEN** 筛选条显示“全部”、“已完成”、“未完成”；显示答题进度条。

### Requirement: 答题详情逻辑
系统 MUST 在作业详情页根据用户角色提供差异化流程，并保持题号状态与 URL 参数同步。

#### Scenario: 学生模式维持现有答题流程
- **WHEN** 学生进入作业详情页
- **THEN** 页面 MUST 继续使用 `courseId`、`homeworkId`、`questionNo` 参数进行题目定位，并支持保存草稿与返回课程作业页。

#### Scenario: 学生模式切换题号
- **WHEN** 学生点击右侧题号索引
- **THEN** 中间区域 MUST 切换到对应题目，且 URL 中 `questionNo` MUST 同步更新。

#### Scenario: 教师模式显示编辑流程操作
- **WHEN** 教师进入作业详情页
- **THEN** 左侧工具栏 MUST 提供保存草稿、添加题目、发布、返回操作，中间区域 MUST 进入可编辑状态。

#### Scenario: 教师发布前二次确认
- **WHEN** 教师点击发布按钮
- **THEN** 系统 MUST 弹窗提示“发布后不可更改”，仅在教师确认后才进入发布状态。

#### Scenario: 发布后锁定详情页
- **WHEN** 作业状态为已发布
- **THEN** 页面 MUST 禁用题目编辑、题目新增、拖拽排序与时间窗修改，并显示不可更改提示。

### Requirement: 教师可新增题目
系统 MUST 允许教师在作业详情页新增题目，并在新增时选择题型与录入基础信息。

#### Scenario: 左侧工具栏触发新增题目
- **WHEN** 教师点击“添加题目”
- **THEN** 系统 MUST 打开新增题目表单，并要求先选择题型再填写题目内容。

#### Scenario: 新增题目后进入可编辑态
- **WHEN** 教师提交新增题目表单且校验通过
- **THEN** 题目 MUST 被追加到对应题型分组，并立即可在中间区域继续编辑。

### Requirement: 教师可编辑题目与参考答案并上传分类型图片
系统 MUST 支持教师编辑题目、参考答案、答案解析，并区分两类图片上传目标。

#### Scenario: 编辑题目与答案内容
- **WHEN** 教师在中间区域修改题目文本、参考答案或答案解析
- **THEN** 系统 MUST 实时更新当前题目草稿状态。

#### Scenario: 上传题目图片
- **WHEN** 教师在题目编辑区域上传图片
- **THEN** 图片 MUST 归类到题目图片集合，不得写入答案解析图片集合。

#### Scenario: 上传答案解析图片
- **WHEN** 教师在答案解析区域上传图片
- **THEN** 图片 MUST 归类到答案解析图片集合，不得写入题目图片集合。

### Requirement: 教师题号排序必须限制在同题型内
系统 MUST 支持教师在右侧题号索引拖拽排序，且仅允许同题型内调整顺序。

#### Scenario: 同题型内拖拽成功
- **WHEN** 教师将某题拖拽到同题型分组内的新位置
- **THEN** 系统 MUST 更新该题型内题目顺序与显示题号。

#### Scenario: 跨题型拖拽被拒绝
- **WHEN** 教师尝试将题目拖拽到其他题型分组
- **THEN** 系统 MUST 阻止排序变更并保持原顺序不变。

### Requirement: 教师必须配置作业开始结束时间
系统 MUST 允许教师定义作业开始时间与结束时间，并在发布前校验时间窗有效性。

#### Scenario: 设置开始结束时间
- **WHEN** 教师在详情页设置开始时间与结束时间，且结束时间晚于开始时间
- **THEN** 系统 MUST 保存该时间窗并在页面展示。

#### Scenario: 发布前时间窗不合法
- **WHEN** 教师点击发布时未设置时间窗或结束时间不晚于开始时间
- **THEN** 系统 MUST 阻止发布并提示教师修正时间配置。

### Requirement: 题型渲染
- **单选/判断**: 题目文本下展示选项列表。
- **多选**: 允许选择多个选项。
- **填空**: 题目下方展示输入文本框。
- **简答**: 使用 `md-editor-rt` 渲染，支持富文本录入与图片。
