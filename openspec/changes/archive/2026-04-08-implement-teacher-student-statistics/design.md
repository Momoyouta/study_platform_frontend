## Context

当前前端已具备课程、作业、学习记录等基础页面与数据流，但角色首页与课程详情缺少可视化统计能力。此次变更会跨越多个模块：`Home`、`CourseDetail`、`HomeworkDetail`、`store`、`http` 与类型定义层；并引入新的图表依赖 `@ant-design/plots`。项目使用 React + MobX + Ant Design + Less，需要在不破坏既有路由与权限行为的前提下，把统计面板按教师/学生角色落地到高频入口。

## Goals / Non-Goals

**Goals:**
- 在教师端首页提供待办工作台（待批改数量）并支持筛选参数透传。
- 在教师课程详情提供课程运营分析（课时学习漏斗、学习频次）卡片化视图。
- 在作业概览提供考情分析（成绩分布、单题正确率、提交状态）并替换旧统计网格。
- 在学生首页提供个人学习中心与 Deadline 追踪面板。
- 在学生课程详情提供学业档案与错题回顾卡片。
- 统一统计接口访问方式、加载态/空态处理与图表渲染约束。

**Non-Goals:**
- 不改动后端接口结构、鉴权策略与角色模型。
- 不覆盖管理员与学校管理员统计模块。
- 不进行通用页面重构或设计系统全面改版。
- 不新增独立移动端路由，仅适配现有布局下的响应式展示。

## Decisions

### Decision 1: 新建统计 API 访问层并集中维护 DTO 映射
- 选择：在现有请求封装下新增 teacher/student statistics API 模块，并同步补充 `src/type` 中的响应类型。
- 原因：统计接口数量多、参数结构相似，集中封装可减少页面层重复拼参和类型漂移。
- 备选方案：
  - 直接在页面组件内调用通用 `http` 方法。
  - 放弃原因：会造成同类接口分散、错误处理和空值处理重复实现。

### Decision 2: 使用“页面容器 + 统计卡片组件”组合替换原区域，而不是新增独立路由
- 选择：保留现有路由结构，按角色替换首页与课程详情中的局部区域。
- 原因：变更范围聚焦，能复用已有权限守卫、面包屑、上下文参数。
- 备选方案：
  - 新建 `/teacher/statistics`、`/student/statistics` 独立页面。
  - 放弃原因：会引入导航分叉，且与用户提出的“替换现有页面内容”不一致。

### Decision 3: 图表能力统一基于 @ant-design/plots，并建立配置适配层
- 选择：使用 `@ant-design/plots` 渲染漏斗/柱状/分布类图表，页面仅传标准化数据。
- 原因：满足需求指定依赖，减少手写图表逻辑，提升可维护性。
- 备选方案：
  - 继续使用纯 Antd 组件 + 自绘统计条。
  - 放弃原因：复杂统计（分桶、正确率）可视化表达弱，可扩展性差。

### Decision 4: 课程与作业统计筛选参数采用可选透传策略
- 选择：统一支持 `courseId?`、`teachingGroupId?`、`assignmentId?`、`startTime?`、`endTime?`，由页面根据上下文填充。
- 原因：兼容“全局概览”和“课程内视图”两类场景，降低接口适配复杂度。
- 备选方案：
  - 强制所有接口必须传完整参数。
  - 放弃原因：会导致首页与课程详情在参数来源上耦合过高，且不符合接口定义的可选特性。

## Risks / Trade-offs

- [Risk] 图表依赖引入后可能带来首屏包体增加 → Mitigation: 仅在统计面板页面按需引入图表组件，避免全局入口提前加载。
- [Risk] 多接口并发导致加载闪烁或请求竞态 → Mitigation: 以卡片粒度独立 loading/error/empty，使用参数签名保证请求结果对应当前筛选条件。
- [Risk] 旧 UI（如 type-stats-grid）移除可能影响用户习惯 → Mitigation: 在新卡片中保留核心指标等价信息，并通过标题/文案保持语义连续。
- [Risk] 课程详情菜单文案调整可能影响测试用例与定位脚本 → Mitigation: 同步更新菜单 key 常量与 E2E 选择器映射。

## Migration Plan

1. 安装并接入 `@ant-design/plots`，完成基础图表组件验证。
2. 新增统计 API 方法与类型定义，完成单端点连通性自测。
3. 先改教师端：首页待办、课程数据、作业考情三块分阶段替换。
4. 再改学生端：首页学习中心、课程详情学业档案与错题回顾。
5. 联调筛选参数与空态，保证无数据时页面结构稳定。
6. 若出现高风险回归，可通过特性开关或组件级回退恢复旧模块展示。
